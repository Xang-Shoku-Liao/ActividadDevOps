# 🤝 Guía para Colaboradores

¡Gracias por tu interés en contribuir a este proyecto! 

## 📝 Reglas Básicas

1. **Usa el modelo de ramas**: Todo nuevo desarrollo debe partir de `develop`.
2. **Crea ramas descriptivas**: Usa prefijos como `feature/`, `bugfix/` o `hotfix/`.
3. **Verifica el Linter**: Asegúrate de que tus archivos Markdown cumplan con las reglas antes de hacer un PR.

## 🔄 Proceso de Pull Request

* Asegúrate de que los GitHub Actions estén en verde (✅).
* Solicita al menos la revisión de 1 integrante del equipo.
* Una vez aprobado, haz el *merge* a `develop`.
